# 市場分析

## なぜ今か

### 需要側の根拠

総務省「令和7年版情報通信白書」では、日本企業の生成AI業務利用率は55.2%まで伸びている。一方で、生成AI導入に際しての懸念は「効果的な活用方法がわからない」に次いで、「社内情報の漏えい等のセキュリティリスク」「ランニングコスト」「初期コスト」が上位にある。

この懸念は、AIチャットよりもAIエージェントで強くなる。AIエージェントは回答するだけでなく、外部APIや業務システムを実行するため、企業は「AIにどこまで権限を渡すか」を決める必要がある。

### タイミングの根拠

| 時期 | AIエージェントの状況 |
|------|-------------------|
| 2023年 | ChatGPT普及期。エージェントは実験段階 |
| 2024年 | LangGraph、CrewAI、AutoGenなどのフレームワークが普及 |
| 2025年 | OpenAI Agents SDKや各フレームワークがtool approval/human-in-the-loopを標準化し始める |
| 2026年 | 本番運用に向けて、承認UI・監査ログ・権限分離が導入課題になる |
| 2027年〜 | 大手プラットフォームとAIガバナンス製品が整い、単純な承認SDKは差別化しにくくなる |

2026年は、問題が顕在化している一方で、軽量に導入できる日本企業向けの実行権限管理レイヤーはまだ空白がある。

### 需要の見方

市場全体を「生成AIを使う企業」と見ると広すぎる。稟議ゲート（仮）が最初に狙うべき需要は、次のような狭く痛い市場である。

- AIエージェントに顧客連絡、支払い、CRM更新、データ変更を任せたい企業
- 自前のSlack承認Botや手動レビューを作ってしまった小規模開発チーム
- SIerやDX部門で、AIエージェントの本番導入が稟議で止まっているチーム
- 顧客向けSaaSにAI自動実行機能を入れたいが、事故が怖くて封印しているチーム

初期市場は数万社ではなく、国内で数十〜数百社の濃い市場から始まると見るべき。その代わり、痛みが本物なら月5万円以上の支払い余地がある。

## 競合分析

### 直接・隣接競合

| 製品/カテゴリ | 何をするか | 脅威 | 稟議ゲート（仮）の差別化 |
|--------------|-----------|------|------------------|
| HumanLayer | AIエージェント向けhuman-in-the-loop SDK/承認ワークフロー | 2024年時点でかなり直接競合。Python/TypeScript、Slack/SMS/email対応実績がある | 2026年にはCodeLayerへ訴求が寄っており、承認単体の弱さを示す警戒信号として見る。稟議ゲート（仮）は実行権限管理へ寄せる |
| LangGraph / LangChain | interrupt/pause/resume、人間判断をフレームワークに内蔵 | 低レベルの承認メカニズムを奪う | 承認UI、監査ログ、通知運用、業務ルールを提供する |
| OpenAI Agents SDK | tool approval、RunStateでの中断/再開 | OpenAI利用者には承認機構が標準化される | 特定SDKに閉じない運用レイヤー、国内通知、監査、権限分離を提供する |
| Microsoft Copilot Studio / Power Automate | agent flows、Human in the loop、Teams/Outlook承認 | Microsoft 365圏の企業に強い | カスタムAIエージェント、スタートアップ、Microsoft外のシステムに軽く入る |
| ServiceNow AI Control Tower | 大企業向けAIガバナンス、発見、監視、統制 | エンタープライズAI統制で強い | 小規模〜中堅チーム向けに、実行直前の承認・監査へ特化する |
| Safe AI Gateway / Cloudflare AI Gateway / Helicone | LLMリクエストの保護、観測、コスト管理 | AI利用管理の予算を取り合う | LLM呼び出しではなく、AIが外部に実行するアクションを制御する |

### 競合から得られる教訓

「AIエージェントを途中で止める」だけでは、フレームワークやSDKの機能として吸収される。HumanLayerのピボットも、承認ワークフロー単体が独立した大きな事業になりにくい可能性を示す。

したがって稟議ゲート（仮）は、以下に集中する。

- 業務担当者が使える承認UI
- 導入稟議に使える監査ログ
- Slack/LINE WORKS/Teamsなど実務通知チャネル
- AIに本物のAPIキーを渡さない権限分離
- 日本企業の稟議・承認文化に合う運用テンプレート

## 参入判断

### 参入する価値がある理由

- AIエージェントの本番化で、危険な実行を止める需要は増える
- フレームワーク内蔵機能だけでは、ビジネスユーザー向けUI、監査ログ、通知運用は埋まらない
- 大企業向けAIガバナンスは重く、小規模AI開発チームには過剰
- 日本企業向けのLINE WORKS、稟議、監査証跡の文脈はローカル最適化余地がある

### 警戒すべき理由

- 承認SDK単体は強い差別化にならない
- HumanLayerのピボットは、承認ワークフロー単体の事業性に対する警戒信号
- Microsoft/ServiceNowなどの大手が企業内AI統制を取りに来ている
- 顧客が「怖い」と言っても、実際に月額課金するほどの痛みかは未検証

### 結論

価値はある。ただし、次にやるべきことはフル実装ではなく、10社ヒアリングと薄いデモによる需要検証である。検証で「すでに自前承認を作っている」「事故時の損害が大きい」「月5万円以上払える」が確認できた場合に、Phase 1 MVPへ進む。
