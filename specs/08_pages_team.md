# チーム・メンバー管理 ページ仕様

## 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/t/:tenant_id/team` |
| アクセス権限 | Admin+（Viewerは非表示） |
| 目的 | テナントメンバーの招待・ロール管理・削除 |

---

```
┌────────────────────────────────────────────────────────┐

│ メンバー (8名)                                           │
├──────────────┬─────────────┬──────────────────────────┤
│ 名前         │ ロール       │ 操作                       │
├──────────────┼─────────────┼──────────────────────────┤
│ 田中 太郎    │ Owner       │ （変更不可）                │
│ 佐藤 花子    │ Admin       │ [ロール変更 ▼]  [削除]      │

│ 山田 二郎    │ Viewer      │ [ロール変更 ▼]  [削除]      │

└────────────────────────────────────────────────────────┘
```

## メンバーテーブル

| 列名 | 内容 |
|------|------|
| アバター | ユーザーのイニシャルまたはプロフィール画像 |
| 名前 | フルネーム（Googleログインの場合はGoogle表示名） |
| メールアドレス | 登録メールアドレス |
| ロール | バッジ形式で表示 |
| 最終ログイン日時 | 「3日前」のような相対表示 |
| 操作 | ロール変更ドロップダウン・削除ボタン |

---

## ロール変更

ドロップダウンで選択するとその場で即時変更。確認は不要（軽微な操作のため）。

**変更できないケース**

- Owner のロールは変更不可（自分自身も含む）。Ownerの移譲は現在未対応（将来的にサポート予定）
- Admin は別の Admin を Owner に昇格させることはできない
- 自分自身を Approver/Viewer に降格させることはできない

---

## メンバー削除

「削除」ボタンをクリックすると確認ダイアログ:

```
田中 太郎 をチームから削除しますか？

この操作を行うと:
・田中さんはこのチームにアクセスできなくなります
・田中さんが担当承認者になっている承認待ちリクエストは、
  対象プロジェクトのデフォルト承認者に再割り当てされます
  （プロジェクトのデフォルト承認者が未設定の場合はテナントの
  Owner/Admin全員に通知されます）

[キャンセル]  [削除する]
```

## 招待フロー

### 招待ボタン → 招待モーダル

```
┌──────────────────────────────────────┐
│ メンバーを招待する                      │
│                                      │
│ メールアドレス                         │
│ [                              ]    │
│ （複数の場合はカンマ区切りで入力）        │
│                                      │
│ ロール                               │
│ ○ Admin                             │
│ ● Approver  ← デフォルト              │
│ ○ Viewer                            │
│                                      │
│ ロールの説明: 承認リクエストの承認・却下   │
│ ができます。設定の変更はできません。      │
│                                      │
│ [キャンセル]      [招待メールを送る]     │
└──────────────────────────────────────┘
```

### 招待メールの内容（参照: `12_notifications.md`）

- 送信者: 「稟議ゲート（仮）（招待者名の代理）」
- 件名: 「{招待者名} さんから {チーム名} への招待が届きました」
- 有効期限: 7日間
- リンク: `/invite/:token`

### 招待の再送信

- 招待中リストの「再送信」ボタンをクリック
- 新しいトークンを生成し、同じメールアドレスに再送信
- 古いトークンは無効化

### 招待の取消

- 「取消」ボタン → 確認なしで即時取消（取消後に誤ってアクセスしても無効ページが表示される）

## プラン上限への対応

現在のプランでのメンバー上限に達した場合:

```
チームの上限（20名）に達しました。
メンバーを追加するにはプランをアップグレードしてください。

[プランを確認する →]
```

- 「+ 招待」ボタンは非活性化

---

## 空の状態（メンバーが自分だけの場合）

```
👥 まだチームメンバーがいません

チームメンバーを招待して、承認フローを一緒に運用しましょう。
承認者（Approver）として招待すると、AIエージェントの承認が届くようになります。

[+ 最初のメンバーを招待する]
```
