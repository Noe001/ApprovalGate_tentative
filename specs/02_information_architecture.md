```
/ (ルート)

└── /t/:tenant_id/                  テナントルート（ログイン必須）

    │

    │   └── /:request_id            承認リクエスト詳細
    │
    ├── /projects                   プロジェクト一覧（Admin+）
    │   ├── /new                    プロジェクト新規作成

    │       └── /api-keys           APIキー管理
    │
    ├── /rules                      承認ルール一覧（Admin+）
    │   ├── /new                    ルール新規作成
    │   └── /:rule_id/edit          ルール編集
    │
    ├── /analytics                  アナリティクス（Admin+ / Viewer）
    │
    ├── /team                       チーム・メンバー管理（Admin+）
    │
    └── /settings                   設定（Admin+）

        ├── /billing                プラン・請求（Ownerのみ）
        └── /account                アカウント個人設定（全ロール）
```

## サイドバーナビゲーション

### Owner / Admin 向け

```
┌─────────────────────┐
│ 🔴 稟議ゲート（仮）        │  ← ロゴ・テナント名
├─────────────────────┤
│ 📊 ダッシュボード     │
│ ✅ 承認リクエスト  [3]│  ← バッジ：承認待ち件数
│ 📁 プロジェクト      │
│ ⚙️  ルール設定       │
│ 📈 アナリティクス    │
│ 👥 チーム           │
├─────────────────────┤
│ ⚙️  設定             │
└─────────────────────┘
```

### Approver 向け（表示を絞る）

```
┌─────────────────────┐
│ 🔴 稟議ゲート（仮）        │
├─────────────────────┤
│ 📊 ダッシュボード     │
│ ✅ 承認リクエスト  [3]│
├─────────────────────┤
│ ⚙️  設定（個人のみ）  │
└─────────────────────┘
```

**注意**: Approver はプロジェクト・ルール・アナリティクスのメニューが非表示になる。ダッシュボードには自分の担当リクエスト件数のみが表示され、テナント全体の統計は表示されない。

## URLパターン設計

| パス | 説明 |
|------|------|
| `/t/{tenant_id}/` | テナント識別子はULID形式（例: `01H...`） |
| `/t/{tenant_id}/approvals` | 承認リクエスト一覧 |
| `/t/{tenant_id}/approvals/{request_id}` | 承認リクエスト詳細 |
| `/t/{tenant_id}/projects` | プロジェクト一覧 |
| `/t/{tenant_id}/projects/{project_id}` | プロジェクト詳細 |
| `/t/{tenant_id}/rules` | ルール一覧 |
| `/t/{tenant_id}/analytics` | アナリティクス |
| `/t/{tenant_id}/team` | チーム管理 |
| `/t/{tenant_id}/settings/general` | 設定（基本） |

### URL設計の方針

- テナントIDをURLに含めることで、ブックマーク・リンク共有が可能
- `/t/` プレフィックスで認証済み領域を明示
- 深いネストは最大3階層まで（可読性維持）

**複数テナント所属時のデフォルト遷移**

ユーザーが複数テナントに所属している場合、ログイン後は**最後にアクセスしたテナント**のダッシュボードに遷移する（`localStorage` に最終アクセス tenant_id を保存）。初回ログイン時は作成したテナントに遷移する。

### 権限のないURLへのアクセス

- Approver が `/t/{tenant_id}/rules` などAdmin専用ページに直接アクセスした場合: **「Not Found」画面を表示**（403ではなく404相当）。リソースの存在を示さない
- 未ログインでアクセスした場合: ログイン画面にリダイレクト。ログイン後は元のURLに戻る
- 他テナントのURLにアクセスした場合: 「Not Found」画面を表示

### エラーページ

| 状況 | 表示 |
|------|------|
| 存在しないURL | 「ページが見つかりません」＋ダッシュボードへのリンク |
| 権限不足 | 「ページが見つかりません」（Not Foundと同一画面） |
| サービス障害 | 「現在メンテナンス中です。しばらくしてから再度アクセスしてください。」 |

詳細は `16_system_behavior.md` のエラーページ仕様を参照。

## ナビゲーション動作

### ページ遷移

- サイドバーのアクティブ状態はURL変化に追従
- ページ遷移時はローディングインジケーターをトップバーに表示
- ブラウザの戻る・進むボタンで正常に動作する

### パンくずナビ

```
ダッシュボード > プロジェクト > my-agent > APIキー
```

- 各セグメントはクリックで該当ページに遷移
- 最後のセグメント（現在地）はリンクにしない

### トップバー

```
┌─────────────────────────────────────────────────────┐
│ [ロゴ]  稟議ゲート（仮） / Acme Corp    [🔔 3]  [👤 田中]  │
└─────────────────────────────────────────────────────┘
```

| 要素 | 内容 |
|------|------|
| ロゴ | クリックでダッシュボードに遷移 |
| テナント名 | クリックでテナント切り替えメニューを表示 |
| 通知ベル | 未読通知件数バッジ表示。クリックで通知ドロワーを開く |
| アカウント | クリックでアカウントメニュー（プロフィール・設定・ログアウト）を表示 |

### テナント切り替え

- 複数テナントに所属するユーザー向け
- テナント名クリック → ドロップダウンで所属テナント一覧表示
- テナントを選ぶとそのテナントのダッシュボードに遷移
- **モバイルでのテナント切り替え**: トップバーのテナント名をタップするとドロップダウンが表示される（ボトムナビには含めない）

## モバイル向けナビゲーション

### ボトムナビゲーション（モバイルのみ）

```
┌────────────────────────────────────┐
│  📊       ✅ [3]    📊      ⚙️      │
│  ホーム   承認     統計    設定     │
└────────────────────────────────────┘
```

- 表示項目はロールに応じて変化（Approverは「承認」のみ）
- バッジは承認待ち件数を常時表示

### モバイル専用UI

- 承認リクエストカードはスワイプ操作に対応（右スワイプ→承認、左スワイプ→却下）
- 確認ダイアログを必ず挟む
