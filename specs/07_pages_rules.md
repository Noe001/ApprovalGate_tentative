# 承認ルール設定 ページ仕様

## 承認ルール一覧

### 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/t/:tenant_id/rules` |
| アクセス権限 | Admin+ |
| 目的 | 承認の自動化・条件分岐・エスカレーションのルールを管理する |

---

### ルールの概念

ルールは「条件 → アクション」の形式で設定する。リクエストが届いた時、ルールを上から順に評価し、最初に一致したルールのアクションを実行する。どのルールにも一致しない場合はデフォルト動作（設定で指定）が適用される。

**重要**: ルールはリクエスト受信時点で評価が確定する。受信後にルールを変更しても、進行中のリクエストには影響しない。

```
条件: アクション名が「DB削除」を含む
  → アクション: 管理者にエスカレーション

条件: メタデータ「amount」が 5000 以下
  → アクション: 自動承認

条件: 送信先メールが「@external.com」
  → アクション: 承認者「佐藤」に通知

（どれも一致しない場合 → デフォルト: 承認者「田中」に通知）
```

---

### レイアウト構成

```
┌───────────────────────────────────────────────────────┐
│ 承認ルール              [テンプレートから追加]  [+ 新規作成] │
├───────────────────────────────────────────────────────┤
│ ℹ️ ルールは上から順に評価されます。ドラッグで順序を変更できます │
├───────────────────────────────────────────────────────┤
│ ≡ 1. DB操作は管理者へ   ✅有効   [編集] [複製] [削除]   │
│ ≡ 2. 少額は自動承認     ✅有効   [編集] [複製] [削除]   │
│ ≡ 3. 外部送信は佐藤へ   ⬜無効   [編集] [複製] [削除]   │
├───────────────────────────────────────────────────────┤
│ デフォルト動作（どのルールにも一致しない場合）             │
│  → 承認者「田中 太郎」に通知               [変更する]   │
└───────────────────────────────────────────────────────┘
```

---

### ルールカードの内容

| 要素 | 内容 |
|------|------|
| ドラッグハンドル（≡） | ドラッグアンドドロップでルールの順序を変更 |
| 順序番号 | ルールが評価される順番 |
| ルール名 | 設定時に付けた名前 |
| 有効/無効バッジ | 緑（有効）/ グレー（無効）。クリックで切り替え可能 |
| 編集ボタン | ルール編集ページに遷移 |
| 複製ボタン | 同じ設定のルールをコピーして作成（名前に「のコピー」を付加） |
| 削除ボタン | 確認ダイアログ表示後に削除 |

---

### テンプレートから追加

よく使われるルールパターンをワンクリックで追加できる。

| テンプレート名 | 内容 |
|--------------|------|
| 高額アクションを承認 | `amount` メタデータが指定金額を超えた場合に承認を要求 |
| キーワードで承認を要求 | アクション名に「削除」「送信」等のキーワードが含まれる場合 |
| 外部ドメインへの送信を承認 | メール宛先が社内ドメイン以外の場合 |
| 深夜・休日は自動承認しない | 平日9〜18時以外は承認要求 |
| 特定プロジェクトを別承認者に | プロジェクト名で承認者を振り分け |

テンプレートを選ぶとルール作成フォームがテンプレートの値で事前入力された状態で開く。

### 条件設定

#### 条件の対象

| 対象 | 説明 |
|------|------|
| アクション名 | SDK/Adapter/APIの `reason` パラメータの値 |
| プロジェクト名 | リクエストが来たプロジェクト名 |
| メタデータキー | SDK/Adapter/APIの `metadata` パラメータの任意キー（例: `amount`） |
| 時刻 | リクエストが届いた時刻（曜日・時間帯） |

#### 条件の演算子

| 対象の型 | 演算子 |
|---------|--------|
| テキスト | 含む / 含まない / 完全一致 / 前方一致 |
| 数値 | より大きい / 以上 / より小さい / 以下 / 等しい |
| 時刻 | 〜時〜分から〜時〜分の間 / 曜日が〜 |

**対象フィールドを選択すると型が自動判定され、入力フォームが切り替わる:**

| 型 | 入力UI |
|----|--------|
| テキスト | テキストフィールド |
| 数値（`amount` 等） | 数値スピナー（小数点2桁まで） |
| 時刻 | 時刻ピッカー（HH:MM形式） |
| 曜日 | チェックボックス（月〜日） |

`metadata.{任意キー}` を条件にする場合、型はユーザーが「テキスト / 数値」から選択する。

#### AND / OR 条件

- 「＋AND条件」: 現在の条件グループにAND条件を追加
- 「＋OR条件」: 独立した条件グループを追加（グループ間はOR評価）
- **評価の優先順位**: グループ内のAND条件をまず評価し、グループ間をORで結ぶ。例: `(A AND B) OR (C AND D)` の形式。AND・ORが混在する場合も括弧で視覚的にグルーピングして表示する

---

### アクション設定

| アクション | 詳細設定 |
|-----------|---------|
| 自動承認する | 追加設定なし |
| 承認者に通知する | 承認者をテナントメンバーから選択（複数選択可能。複数の場合は全員に通知し、誰か1人の承認で完了） |
| 自動却下する | 却下理由テキストを設定可能（AIエージェントへのコールバックに含まれる） |
| エスカレーション | エスカレーション先のメンバーを選択・エスカレーション前の待機時間を設定。**循環エスカレーション（AからBへ、BからAへ戻る設定）はシステムが検出してエラーを表示する** |

---

### ルールのプレビュー・テスト

「プレビュー・テスト」ボタンを押すと、サイドパネルが開く。

```
テストデータを入力してください:

アクション名:  [                    ]
メタデータ:   key [       ] value [       ]  [＋追加]

[このルールを評価する]

────────────────────
結果: ✅ このルールが適用されます
アクション: 承認者「田中」に通知
```

- 実際にリクエストは送信されない（ドライラン）
- 一致しない場合は「このルールは適用されません（条件X が一致しませんでした）」と表示

## デフォルト動作の設定

ルール一覧ページ下部の「デフォルト動作」セクション。

| 設定項目 | 内容 |
|---------|------|
| デフォルト承認者 | どのルールにも一致しない場合に通知するメンバー（必須） |
| デフォルトタイムアウト | テナント全体のデフォルトタイムアウト時間（分）。プロジェクト設定で上書き可能 |
| タイムアウト時のデフォルト動作 | 拒否（推奨）/ 承認 |
