# 承認リクエスト ページ仕様

## 承認リクエスト一覧

### 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/t/:tenant_id/approvals` |
| アクセス権限 | 全ロール（表示範囲はロールで異なる） |
| 目的 | 承認リクエストの確認・承認・却下・履歴参照 |

### フィルター・検索

| フィルター | 選択肢 |
|-----------|--------|
| ステータス | すべて / 承認待ち / 承認済み / 却下 / タイムアウト / 自動承認 |
| プロジェクト | テナント内の全プロジェクト一覧から選択 |
| 承認者 | テナント内の全メンバーから選択（Admin+のみ） |
| 期間 | 今日 / 過去7日間 / 過去30日間 / カスタム範囲 |

- キーワード検索: アクション名・理由・メタデータに対して部分一致検索
- URLのクエリパラメータにフィルター状態が反映される（共有・ブックマーク可能）
- デフォルト: ステータス「承認待ち」・期間「今日」

---

### テーブル列定義

| 列名 | 内容 | ソート |
|------|------|-------|
| チェックボックス | 一括操作用の選択 | - |
| ステータス | アイコン + テキスト | ✓ |
| アクション名 | SDK/Adapter/APIで指定した `reason` 文字列 | ✓ |
| プロジェクト | プロジェクト名（クリックでプロジェクト詳細に遷移） | ✓ |
| 担当承認者 | 割り当てられた承認者名。削除済みユーザーの場合は「削除済みユーザー」と表示（グレーアウト） | ✓ |
| 受信日時 | リクエストが届いた日時 | ✓（デフォルト: 新しい順） |
| 経過時間 | 承認待ちの場合は経過時間、完了済みは応答時間 | - |
| 操作 | 詳細を見るボタン（[▶]）/ 承認ボタン / 却下ボタン（承認待ちのみ） |

---

### テーブル行のインタラクション

- 行全体クリック → 承認リクエスト詳細ページに遷移
- 「承認」ボタンクリック → その場で即時承認（確認なし）。行のステータスが更新される
- 「却下」ボタンクリック → 却下理由入力モーダルを表示

---

### 一括操作

チェックボックスで複数選択後に表示されるアクションバー:

```
5件を選択中  [一括承認]  [一括却下]  [選択解除]
```

- 一括承認: 「選択した5件を承認しますか？」確認ダイアログ → OK で全件承認
- 一括却下: 却下理由を1つ入力 → 全件に同じ理由で却下
- 異なるステータスが混在する場合: 「承認待ち3件のみ処理します（承認済み2件はスキップ）」と内訳を表示してから確認ダイアログを表示。処理対象が0件の場合はボタン自体を非活性化

---

### 空の状態

```
✅ 承認待ちのリクエストはありません
   AIエージェントがSDK/Adapter/APIを通じてアクションを試みると、ここに表示されます。
   [統合ガイドを確認する →]
```

## 承認リクエスト詳細

### 基本情報

| 項目 | 内容 |

| 目的 | 個別リクエストの詳細確認・承認・却下・履歴 |

### アクション情報パネル

| 要素 | 内容 |
|------|------|
| ステータスバッジ | 現在のステータスを色付きバッジで表示 |
| タイムアウトカウントダウン | 承認待ち中のみ表示。タイムアウトまでの残り時間をリアルタイム更新 |
| アクション名 | SDK/Adapter/APIの `reason` パラメータで指定された文字列 |
| 理由・説明 | SDK/Adapter/APIの `description` パラメータで指定された詳細説明 |
| メタデータ | SDK/Adapter/APIの `metadata` パラメータで渡された任意のキー/値ペア |
| プロジェクト名 | リンクになっており、クリックでプロジェクト詳細に遷移（Admin+のみ） |
| 担当承認者 | このリクエストに割り当てられている承認者名 |
| リクエストID | コピーボタン付きで表示（サポートへの問い合わせ等で使用） |

---

### 処理履歴タイムライン

リクエストの開始から終了までのイベントを時系列で表示。

| イベント | 表示内容 |
|---------|---------|
| リクエスト受信 | 受信日時 |
| 通知送信 | 送信先（Phase 1はSlack、LINE WORKS/Teamsは検証後）・送信先ユーザー名 |
| 通知失敗 | エラー内容（チャンネルが見つからない等） |
| エスカレーション | エスカレーション先ユーザー名 |
| 承認 | 承認者名・承認日時 |
| 却下 | 却下者名・却下理由・却下日時 |
| タイムアウト | タイムアウト処理内容（デフォルト拒否/承認） |
| 自動承認 | 適用されたルール名 |

---

### 承認・却下操作

#### 承認ボタン

- ページ右上とアクション情報パネル下部の2カ所に配置
- クリック後: ステータスが「承認済み」に更新される。確認ダイアログは不要
- 承認済み・却下済み・タイムアウト済みの場合はボタンを非活性化

#### 却下ボタン

- クリックで却下理由入力モーダルを表示

**却下モーダル**

```
タイトル: リクエストを却下しますか？

却下理由（任意）:
[                              ]
例: 対象リストに誤りがあります

[キャンセル]  [却下する]
```

- 却下理由は任意入力（空でも却下可能）・最大500文字
- 入力した却下理由はAIエージェントへのコールバックと処理履歴に記録される

---

### エスカレーション操作

- 承認待ち中のみ表示される「...」メニューから「エスカレーション」を選択
- 「誰にエスカレーションしますか？」→ テナントメンバー一覧から選択
- 選択した人に通知が送られ、処理履歴に記録される
