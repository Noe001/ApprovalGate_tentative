# 設定ページ群 仕様

## 設定のナビゲーション構造

```
/settings/
├── /general        テナント基本設定
├── /notifications  通知設定（Phase 1はSlack、LINE WORKS/Teamsは検証後）
├── /billing        プラン・請求（Ownerのみ）
└── /account        アカウント個人設定（全ロール）
```

設定ページは左側にサブナビゲーションを持つ2ペインレイアウト。

---

## テナント基本設定（`/settings/general`）

### アクセス権限
Admin+

### 設定項目

| 項目 | 説明 |
|------|------|
| チーム名 | テナントの表示名。変更可能 |
| チームID | URLに使用するID。一度設定したら変更不可 |
| タイムゾーン | アナリティクスの時刻表示に使用。デフォルト: Asia/Tokyo |
| デフォルトのタイムアウト時間 | 全プロジェクトのデフォルト値（分）。プロジェクト設定で上書き可能 |
| タイムアウト時のデフォルト動作 | 拒否（推奨）/ 承認 |
| fail-closed設定 | サービス障害時に全リクエストを拒否するか否か（デフォルト: 拒否） |

### 危険ゾーン（ページ最下部）

「テナントを削除する」セクション（赤い枠で囲む）

```
⚠️ 危険な操作

テナントを削除すると、すべてのデータ（プロジェクト・ルール・リクエスト履歴）が
完全に削除されます。この操作は取り消せません。

[テナントを削除する]
```

削除確認ダイアログ:
1. チーム名の入力を求める
2. 「削除後30日以内のデータ復元を希望する場合はサポートへ」の案内

## 通知設定（`/settings/notifications`）

### アクセス権限
Admin+（Slackのテナント全体設定。LINE WORKS/Teamsは検証後に追加）  
全ロール（個人通知先の設定）

### テナント全体の通知設定

#### Slack連携

```
┌──────────────────────────────────────────────────┐
│ Slack連携                               [連携済み ✅] │
│                                                  │
│ ワークスペース: Acme Corp                           │
│ デフォルト通知チャンネル: #ai-approvals              │
│                                              [変更]│
├──────────────────────────────────────────────────┤
│ [テスト通知を送る]  [Slack連携を解除する]              │
└──────────────────────────────────────────────────┘
```

- 「変更」クリック → Slack OAuth フローを再実行
- 「テスト通知を送る」→ デフォルトチャンネルにテスト投稿。投稿内容を画面上にもプレビュー表示
- 「Slack連携を解除する」→ 確認ダイアログ（解除後は通知が届かなくなる旨の警告）

#### LINE WORKS連携（Phase 1.5候補）

Phase 1では実装せず、Waitlist表示または「利用希望を登録する」ボタンを表示する。Phase 0ヒアリングで10社中3社以上が必須チャネルとして挙げた場合、Slack連携と同様の構成で追加する。

### 個人通知設定

各メンバーが自分の通知先を設定できる。

| 項目 | 内容 |
|------|------|
| Slack通知先 | 個人DM / 指定チャンネルから選択（テナント連携が前提） |
| LINE WORKS通知先 | Phase 1.5以降。個人DM |
| メール通知 | 登録メールアドレスへ。頻度: 即時 / 1時間まとめ / 日次まとめ |
| 通知する条件 | 自分に割り当てられたリクエストのみ / すべてのリクエスト（Admin+のみ選択可） |

## プラン・請求（`/settings/billing`）

### アクセス権限
Ownerのみ

### 現在のプラン表示

```
┌────────────────────────────────────────────────────────┐
│ 現在のプラン: スタンダード                  [プランを変更する] │
├────────────────────────────────────────────────────────┤
│ 利用状況                                               │
│ メンバー: 8 / 20名                                     │
│ 今月のリクエスト: 1,204 / 5,000件                       │
│ ████████░░░░░░░░░░░░  24%使用中                       │
│                                                        │
│ 次回更新日: 2026年06月01日                              │
│ 月額: ¥50,000（税込）                                  │
└────────────────────────────────────────────────────────┘
```

### プラン変更

プラン比較テーブルを表示後、変更するプランを選択。

| | スターター | スタンダード | Pro |
|-|-----------|------------|-----|
| 月額 | ¥15,000 | ¥50,000 | ¥150,000 |
| メンバー | 5名 | 20名 | 無制限 |
| リクエスト/月 | 500件 | 5,000件 | 無制限 |
| クレデンシャル保管（Level 2） | ✗ | Phase 2で限定用途から検証 | ✓ |
| カスタムワークフロー | ✗ | ✗ | ✓ |
| SLA | ✗ | ✗ | 99.9% |

- アップグレード: 即時反映・差額を日割り計算
- ダウングレード: 次回更新日から反映

### 支払い方法

| 項目 | 内容 |
|------|------|
| 登録カード | カード番号の末4桁・有効期限を表示 |
| 変更ボタン | Stripe の決済フォームを開く |
| 請求先メール | 請求書の送付先を変更可能 |

### 請求書一覧

| 列 | 内容 |
|----|------|
| 請求日 | 日付 |
| 金額 | 税込金額 |
| ステータス | 支払済み / 未払い |
| ダウンロード | PDF請求書をダウンロード |

### リクエスト上限超過時

上限の80%到達時: バナー通知「今月のリクエスト数が上限に近づいています」  
上限到達時: 追加リクエストは受け付けられず、SDKに `QUOTA_EXCEEDED` エラーが返る  
超過分の従量課金オプション（Pro以外は選択制）:「超過分を ¥5/件 で自動追加する」トグル

## アカウント設定（`/settings/account`）

### アクセス権限
全ロール（自分のアカウント情報のみ変更可能）

### 設定項目

| 項目 | 内容 |
|------|------|
| 表示名 | 承認履歴・通知に表示される名前 |
| プロフィール画像 | アップロードまたはURL指定。未設定の場合はイニシャルアバター |
| メールアドレス | 変更すると確認メールを再送信 |
| パスワード変更 | 現在のパスワード入力 → 新しいパスワード設定 |
| 2段階認証 | TOTP（Google Authenticator等）の設定・解除 |
| 言語設定 | 日本語 / English（現時点は日本語のみ。将来対応予定） |

### セッション管理

```
アクティブなセッション:
  Chrome on Windows  (現在のセッション)   東京    ─
  Safari on iPhone   3日前ログイン        東京    [ログアウト]
  Firefox on Mac     7日前ログイン        大阪    [ログアウト]

  [すべてのセッションからログアウト（現在除く）]
```

### アカウント削除

- 「アカウントを削除する」ボタン（ページ最下部）
- チームの Owner の場合は別のメンバーにオーナー権限を移譲してから削除するよう案内
- 削除後30日はデータを保持（誤操作からの復元期間）
