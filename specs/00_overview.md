## プロダクト概要

**稟議ゲート（仮）**は、AIエージェントの危険な実行を、人間承認・監査ログ・権限分離で本番運用できるようにする実行権限管理インフラ。

SDKは導入手段の1つであり、差別化の本体ではない。LangGraphやOpenAI Agents SDKのネイティブinterrupt/approvalを補完し、業務担当者が使える承認UI、監査証跡、通知運用、将来的なクレデンシャル分離を提供する。

### 仕様書の位置づけ

本ディレクトリの仕様はPhase 2以降を含むプロダクトビジョンまで記述している。Phase 1 MVPで実装する範囲は [19_mvp_scope.md](19_mvp_scope.md) を正とする。

### 登場する主体

```
[AIエージェント（顧客のシステム）]
        ↓ SDK/Adapter/API経由でリクエスト
[稟議ゲート（仮） API]
        ↓ 通知
[承認者（人間）] ← Slack / Webダッシュボード（LINE WORKS/Teamsは検証後）
        ↓ 承認 or 却下
[AIエージェント] ← 結果を受け取り実行再開 or 停止
```

## 用語定義

| 用語 | 定義 |
|------|------|
| **テナント** | 稟議ゲート（仮）を契約している企業・組織の単位。データは完全にテナント間で分離される |
| **プロジェクト** | テナント内で稟議ゲート（仮）に接続するAIエージェントアプリの単位。1アプリ = 1プロジェクト |
| **APIキー** | プロジェクトを識別するためのシークレットキー。SDK/Adapter/API連携の設定に使用する |
| **承認リクエスト** | AIエージェントが起こそうとしたアクションの承認依頼。1回のアクション = 1リクエスト |
| **アクション** | AIエージェントが実行しようとする操作（メール送信・支払い・データ変更など） |
| **ルール** | 承認を自動化または条件分岐させるための設定。「金額が¥5,000以上なら承認必要」など |
| **承認者（Approver）** | 承認リクエストを受け取り、承認または却下する権限を持つユーザー |
| **管理者（Admin）** | テナントの設定・メンバー管理・ルール設定を行う権限を持つユーザー |
| **オーナー（Owner）** | テナントの最上位権限。課金・テナント削除を含む全操作が可能 |
| **fail-closed** | 稟議ゲート（仮）のサービスが利用不能な場合、アクションをデフォルトで「拒否」する設計方針 |
| **タイムアウト** | 承認リクエストが一定時間内に処理されなかった場合の自動処理 |
| **エスカレーション** | 承認者が応答しない場合に上位の承認者に通知を転送する仕組み |
| **クレデンシャル保管** | 実際の外部APIキーを稟議ゲート（仮）が保管し、代理実行するセキュリティ機能（Level 2）。Phase 2で限定用途から実装する |

## 設計原則

### 1. チュートリアル不要のUX

- **空の状態には必ず次のアクションを示す**（"まだリクエストがありません。統合ガイドを確認してください" + ボタン）
  - Approverの空状態は「担当リクエストが届くまでお待ちください」と表示（操作アクションなし）
- **用語は画面内で説明する**（アイコン横の「?」ホバーで説明表示）
- **重要な操作は3クリック以内**で完了できる
- **操作の結果は即座にフィードバック**する（ローディング・成功・エラーの明示）
- **破壊的操作は必ず確認ダイアログ**を挟む
- **フィルターがかかっている状態で0件の場合**は「フィルターを外すと〇件あります」と表示（"存在しない"と"フィルター済み"を区別できるようにする）

### 2. Slack主体の設計

- 承認者はSlackだけで全ての承認操作が完結する
- Webダッシュボードは「管理・確認・高度な操作」のための補完的な場所
- Slack通知の内容は「承認判断に必要な情報がすべて揃っている」設計

### 3. ロール別の情報制御

- 承認者は自分に割り当てられたリクエストのみ見える
- 管理者は全プロジェクトの設定・統計が見える
- 権限外の画面・機能はメニューから非表示（エラーではなく存在しない）

### 4. マルチテナント分離

- テナント間のデータは完全に分離される
- URLにテナント識別子を含める（例: `app.ringigate.com/t/{tenant_id}/`）
- テナント設定はオーナーのみが変更できる

## 画面共通仕様

### レイアウト構造

デスクトップ・タブレット:
```
┌──────────────────────────────────────────────┐
│ トップバー（ロゴ・テナント名・通知・アカウント）     │
├──────────┬───────────────────────────────────┤
│          │ パンくずナビ                         │
│ サイド    ├───────────────────────────────────┤
│ バー      │                                   │
│ ナビ      │   メインコンテンツ                   │
│          │                                   │
│          │                                   │
└──────────┴───────────────────────────────────┘
```

モバイル（767px以下）:
```
┌──────────────────────────────────────────────┐
│ トップバー（ロゴ・通知・アカウント）               │
├──────────────────────────────────────────────┤
│                                              │
│   メインコンテンツ（フルスクリーン）             │
│                                              │
├──────────────────────────────────────────────┤
│      ボトムナビゲーション                       │
└──────────────────────────────────────────────┘
```

### カラー定義（ステータス）

| ステータス | 色 | 意味 |
|-----------|-----|------|
| 承認待ち | 黄（Amber） | 人間の判断待ち |
| 承認済み | 緑（Green） | 実行許可済み |
| 却下 | 赤（Red） | 実行停止 |
| タイムアウト | グレー（Gray） | 時間切れで自動処理済み |
| 自動承認 | 青（Blue） | ルールにより自動的に承認 |
| エラー | 濃赤（Dark Red） | システムエラー |

### レスポンシブ対応

- デスクトップ（1280px以上）: 左サイドバー + メインコンテンツ
- タブレット（768〜1279px）: サイドバー折りたたみ可
- モバイル（767px以下）: ボトムナビゲーション + フルスクリーンコンテンツ
