## プロジェクト一覧

### 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/t/:tenant_id/projects` |
| アクセス権限 | Admin+ |
| 目的 | AIエージェントアプリ（プロジェクト）の管理・新規作成 |

---

### レイアウト構成

```
┌──────────────────────────────────────────────────────┐
│ プロジェクト                          [+ 新しいプロジェクト] │
├──────────────────────────────────────────────────────┤
│ 🔵 marketing-agent    承認待ち 3件   月47件   [→]     │
│ 🔵 billing-agent      承認待ち 0件   月12件   [→]     │
│ ⚫ cleanup-agent（無効）承認待ち 0件  月 0件   [→]     │
└──────────────────────────────────────────────────────┘
```

---

### プロジェクトカードの内容

| 要素 | 内容 |
|------|------|
| ステータスドット | 🔵 アクティブ / ⚫ 無効 |
| プロジェクト名 | 作成時に設定した名前 |
| 承認待ち件数 | 現在の承認待ちリクエスト数。クリックで承認一覧（そのプロジェクトでフィルター済み）に遷移 |
| 今月の処理件数 | 当月の累計処理件数 |
| 詳細ボタン [→] | プロジェクト詳細ページに遷移 |

---

## プロジェクト新規作成（`/t/:tenant_id/projects/new`）

### 入力フォーム

| フィールド | 種別 | バリデーション |
|-----------|------|--------------|
| プロジェクト名 | テキスト | 必須・2〜50文字 |
| 説明 | テキストエリア | 任意・〜200文字 |
| デフォルト承認者 | メンバー選択（複数可） | 任意 |
| タイムアウト設定 | 数値（分） | 任意・デフォルト30分 |
| タイムアウト時の動作 | セレクト | 拒否（推奨） / 承認 |

**作成ボタン**

「プロジェクトを作成する」→ APIキー発行画面に遷移

**デフォルト承認者未設定の警告**

デフォルト承認者を設定せずに保存しようとした場合、確認バナーを表示する:

```
⚠️ デフォルト承認者が設定されていません。
どのルールにも一致しないリクエストが承認されなくなります。
[承認者を設定する] または [スキップして後で設定する]
```

詳細は `16_system_behavior.md` セクション3「デフォルト承認者が未設定の場合の動作」を参照。

### タブ: APIキー（`/t/:tenant_id/projects/:project_id/api-keys`）

#### 画面構成

```
┌──────────────────────────────────────────────────────┐
│ APIキー                               [+ 新しいキーを発行] │
├──────────────────────────────────────────────────────┤
│ 本番キー   rg_live_****...****  作成: 2026/01/15  [失効] │
│ テストキー  rg_test_****...****  作成: 2026/01/15  [失効] │
└──────────────────────────────────────────────────────┘
```

#### APIキーの種別

| 種別 | プレフィックス | 用途 |
|------|--------------|------|
| 本番キー | `rg_live_` | 実際の承認フローを動かす |
| テストキー | `rg_test_` | 開発・テスト用。実際の通知は送られない |

#### APIキー発行フロー

1. 「+ 新しいキーを発行」ボタン
2. モーダル: キー名（任意）・種別（本番/テスト）を選択
3. 「発行する」ボタン
4. **発行直後のみ** キーの全文を表示するモーダル

```
✅ APIキーを発行しました

rg_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxx   [コピー]

⚠️ このキーは一度しか表示されません。必ずコピーして安全な場所に保管してください。
後から確認することはできません。

[コピーしました・閉じる]
```

5. 閉じるとキーはマスク表示（`rg_live_****...abc123`）のみ

#### APIキーの失効

- 「失効」ボタンをクリック
- 確認ダイアログ: 「このキーを使用しているSDK/Adapter/API連携が動作しなくなります。よろしいですか？」
- 失効は即時反映。元に戻せない

**ダウンタイムなしでのローテーション手順は `16_system_behavior.md` セクション7「APIキーローテーション手順」を参照。**

### タブ: ルール

このプロジェクト専用のルール設定。テナント全体のルールより優先される。
詳細は `07_pages_rules.md` 参照。プロジェクト詳細内に同じUIをインライン表示。

---

### タブ: 設定

| 設定項目 | 内容 |
|---------|------|
| プロジェクト名 | 変更可能 |
| 説明 | 変更可能 |
| デフォルト承認者 | このプロジェクトの承認リクエストがデフォルトで誰に届くかを設定 |
| タイムアウト時間 | 分単位で設定（0=タイムアウトなし） |
| タイムアウト時の動作 | 拒否 / 承認 |
| プロジェクトの無効化 | トグル。無効中はSDK/Adapter/APIからのリクエストを受け付けない。承認待ちリクエストは保持 |
| プロジェクトの削除 | 赤いボタン。確認ダイアログ（プロジェクト名の入力）を経て実行。承認待ちリクエストは即時タイムアウト扱い（`TIMED_OUT`）になり、完了済みリクエスト・ログは30日後に物理削除される（`16_system_behavior.md` セクション2参照） |

**削除確認ダイアログ**

```
⚠️ プロジェクトを削除しますか？

この操作は取り消せません。
「marketing-agent」と入力して削除を確定してください。

[                    ]

[キャンセル]  [削除する（取り消し不可）]
```
