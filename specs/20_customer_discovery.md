# 顧客ヒアリング計画

## 目的

実装前に、稟議ゲート（仮）が解くべき痛みが本物かを確認する。特に「怖いので欲しい」ではなく、「既に自前対応している」「本番導入が止まっている」「有料でも解決したい」状態を探す。

---

## 対象セグメント

| 優先度 | セグメント | 探す相手 |
|--------|------------|----------|
| 高 | AI SaaS / AIスタートアップ | CTO、PdM、AIエージェント開発者 |
| 高 | AI機能を持つ小規模SaaS | 創業者、リードエンジニア |
| 中 | SIer新規事業 / DX支援 | 事業責任者、プリセールス、PM |
| 中 | 社内AIエージェント導入チーム | DX推進、情報システム、業務部門PM |
| 低 | 大企業AIガバナンス部門 | まずは学習目的。初期販売対象にしない |

---

## ヒアリングで確認する仮説

| 仮説 | 確認したいこと |
|------|----------------|
| 実行権限が本番化の障害になっている | AIにメール送信、DB更新、支払いなどを任せることが怖いか |
| 自前承認の負担がある | Slack bot、手動レビュー、独自ワークフローを作っているか |
| 監査ログが稟議に効く | 誰が何を承認したかを後から説明する必要があるか |
| LINE WORKS需要がある | Slack非採用企業でLINE WORKSが導入条件になるか |
| Level 2が差別化になる | AIに本物のAPIキーを渡さないことに価値を感じるか |
| 支払い意思がある | 月5万円、または有料PoCに進む理由があるか |

## 質問票

### 1. 現在のAIエージェント利用状況

- どの業務でAIエージェントを使っていますか
- そのエージェントは外部システムに対して何かを実行しますか
- 実行する場合、具体的にはメール送信、CRM更新、DB更新、支払い、広告出稿、ファイル操作のどれに近いですか
- 本番運用中ですか、PoCですか、構想段階ですか

### 2. 止めたい実行とリスク

- AIに任せたいが怖くて止めている操作はありますか
- その操作で事故が起きた場合、何が一番困りますか
- 金額損失、顧客信用、個人情報、業務停止、社内説明のどれが大きいですか
- 現在はどのタイミングで人間が確認していますか

### 3. 既存の代替手段

- すでに承認BotやSlack確認フローを作っていますか
- 作っている場合、どれくらいの期間で作りましたか
- メンテしていて困っていることは何ですか
- 監査ログや承認履歴は残っていますか
- 却下理由やタイムアウト時の扱いはどうしていますか

### 4. 承認者と通知チャネル

- 承認者はエンジニア、業務担当者、管理職、顧客企業の担当者のどれですか
- 普段使っている通知ツールはSlack、Teams、LINE WORKS、メールのどれですか
- Slackだけだと導入できない顧客はいますか
- LINE WORKSがあれば導入可能性は上がりますか
- スマホ承認は必須ですか、それともSlack/ブラウザで十分ですか

### 5. 稟議・顧客説明

- AIが実行する機能を社内稟議や顧客に説明する必要はありますか
- 「誰が何を承認したか」の履歴は必要ですか
- 監査ログがあると本番導入に進みやすくなりますか
- どの形式のログが必要ですか。画面、CSV、API、PDFのどれですか

### 6. Level 2の価値

- AIエージェントに外部APIキーを渡していますか
- AIに本物のAPIキーを渡さず、承認後に稟議ゲート（仮）が代理実行する仕組みに価値を感じますか
- 最初に対応してほしい代理実行は何ですか
- メール送信、支払い、CRM更新、ファイル削除、広告出稿のどれが一番怖いですか

### 7. 支払い意思

- この問題を解決するために月額で支払う予算はありますか
- 月5万円なら高いですか、妥当ですか、安いですか
- 有料PoCならいくらまで通しやすいですか
- 稟議を通すために必要な資料や機能は何ですか
- 導入を決める人は誰ですか

## デモ時に見るポイント

薄いデモを見せながら、以下を観察する。

| 観察項目 | 良い反応 | 悪い反応 |
|----------|----------|----------|
| Slack通知 | 「この情報なら判断できる」 | 「結局ダッシュボードを開かないと無理」 |
| Web監査ログ | 「稟議/顧客説明に使える」 | 「ログは不要」 |
| LINE WORKS Waitlist | 「これがないと導入できない」 | 「Slackで十分」 |
| Level 2モック | 「APIキー分離が重要」 | 「そこまでは要らない」 |
| 価格 | 「月5万円なら通せる」 | 「無料なら使う」 |

## 判定スコア

| 項目 | 0点 | 1点 | 2点 |
|------|-----|-----|-----|
| 実行系AIの有無 | ない | PoC中 | 本番/本番直前 |
| 危険操作の明確さ | 曖昧 | あるが低リスク | 高リスク操作が明確 |
| 既存代替の有無 | ない | 手動運用 | 自前Bot/自前実装あり |
| 監査ログ需要 | ない | あれば便利 | 稟議/顧客説明に必須 |
| 通知チャネル制約 | ない | Slack以外も希望 | LINE WORKS/Teams必須 |
| 支払い意思 | 無料なら | 低額なら | 月5万円以上/有料PoC可 |

合計8点以上をDesign Partner候補とする。
