# ユーザーロールと権限

## ロール一覧

| ロール | 説明 | 付与方法 |
|--------|------|---------|
| **Owner（オーナー）** | テナント作成者。全権限を持つ | テナント作成時に自動付与。変更不可 |
| **Admin（管理者）** | テナントの設定・運用を担当。課金・テナント削除以外の全操作が可能 | Ownerが付与 |
| **Approver（承認者）** | 承認リクエストの承認・却下のみ可能。設定変更不可 | Owner / Adminが付与 |
| **Viewer（閲覧者）** | ダッシュボード・ログの閲覧のみ可能。承認・設定変更不可 | Owner / Adminが付与 |

## ロール設計の補足

### Approverの可視範囲

- Approverは自分が「担当承認者」として設定されているリクエストのみ表示される
- 他の承認者に割り当てられたリクエストは見えない
- サイドバーの「プロジェクト」「ルール」「アナリティクス」メニューは非表示

### 複数ロール

- 1人のユーザーは1テナント内で1ロールのみ持つ（複数ロール不可）
- 複数テナントに所属することは可能（テナントごとに異なるロールを持てる）

### ロール変更の制約

- OwnerのロールをAdminに降格させることはできない（Ownerは常にOwner）
- OwnerはAdminを別のユーザーに移すことはできない（Ownerは1テナントに1人）
- Adminは別のAdminを作れるが、自分を降格させることはできない
